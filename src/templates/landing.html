📤 pooortal

Secure, token-based file upload service powered by tus protocol.

Getting Started

To upload files, you need a valid access token. Contact your administrator to generate one.

Once you have a token, visit:

https://your-domain.com/<your-token>

Features

  • Resumable uploads – pause and resume anytime
  • Token-based authentication
  • Real-time progress tracking
  • Secure path validation

Programmatic Uploads

Create an upload and send data in 100 MB chunks using the tus protocol:

# 1. Create the upload (replace FILE and TOKEN) FILE="path/to/file" TOKEN="your-token" SIZE=$(wc -c < "$FILE") CHUNK=104857600 # 100 MB LOCATION=$(curl -s -D- -X POST https://your-domain.com/upload/ \ -H "Tus-Resumable: 1.0.0" \ -H "Upload-Length: $SIZE" \ -H "Upload-Metadata: token $(echo -n $TOKEN | base64)" \ | grep -i ^location: | tr -d '\r' | cut -d' ' -f2) # 2. Upload in 100 MB chunks OFFSET=0 while [ "$OFFSET" -lt "$SIZE" ]; do BYTES=$(( SIZE - OFFSET < CHUNK ? SIZE - OFFSET : CHUNK )) curl -X PATCH "$LOCATION" \ -H "Tus-Resumable: 1.0.0" \ -H "Upload-Offset: $OFFSET" \ -H "Content-Type: application/offset+octet-stream" \ -H "Content-Length: $BYTES" \ --data-binary @<(dd if="$FILE" bs=1 skip=$OFFSET count=$BYTES 2>/dev/null) OFFSET=$(( OFFSET + BYTES )) done
